# Contributing Guidelines

## Workflow
- Use trunk-based development with feature branches: `feat/<short-description>`
- Make small, focused PRs (<300 LOC)
- Include tests with ≥70% coverage

## Commit Messages
Use conventional commits:
- `feat:` for new features
- `fix:` for bug fixes
- `refactor:` for code refactoring
- `test:` for test changes
- `docs:` for documentation
- `chore:` for maintenance tasks

## PR Checklist
- [ ] Tests pass
- [ ] Coverage ≥70%
- [ ] Linting passes
- [ ] Documentation updated
- [ ] Issue linked
